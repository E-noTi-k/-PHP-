Для того чтобы твой проект заработал на другом компьютере, недостаточно просто скопировать папку. Поскольку это PHP-приложение, ему нужна серверная среда и база данных.

Вот пошаговая инструкция, которую можно вставить в пояснительную записку или использовать самому.

Инструкция по развертыванию проекта на новом устройстве
Шаг 1: Подготовка среды

PHP — это серверный язык, поэтому файлы не откроются двойным кликом. Тебе нужно установить локальный сервер:

Для Windows: Скачай и установи OpenServer (самый удобный) или XAMPP.

Для macOS: Используй MAMP.

После установки запусти модули Apache (веб-сервер) и MySQL (база данных).

Шаг 2: Перенос файлов

Скопируй свою папку с проектом (например, Курсач).

Вставь её в корневую директорию твоего сервера:

В OpenServer: OSPanel\domains\

В MAMP: C:\MAMP\htdocs\ (Windows) или /Applications/MAMP/htdocs/ (Mac).

Шаг 3: Перенос базы данных

Чтобы данные (товары, пользователи, продажи) сохранились, нужно перенести «дамп» базы.

На старом устройстве: Зайди в phpMyAdmin → выбери базу dairy_shop → вкладка Экспорт → нажми «Вперед». Получишь файл dairy_shop.sql.

На новом устройстве: Зайди в phpMyAdmin (обычно localhost/phpmyadmin).

Создай новую базу данных с названием dairy_shop (кодировка utf8_general_ci).

Выбери созданную базу → вкладка Импорт → выбери файл dairy_shop.sql → нажми «Вперед».

Шаг 4: Настройка подключения (config/db.php)

На новом компьютере пароли от MySQL могут отличаться. Открой файл config/db.php и проверь настройки:

Шаг 6: Запуск сайта

Открой браузер и введи адрес:
http://localhost/Курсач/login.php

Дополнительно: Как зайти на сайт с телефона (в одной Wi-Fi сети)

Если ты хочешь показать адаптивность на реальном телефоне или планшете:

Узнай локальный IP-адрес компьютера, на котором запущен сервер (в командной строке введи ipconfig, ищи IPv4-адрес, например 192.168.1.5).

Убедись, что телефон и компьютер подключены к одному Wi-Fi.

На телефоне в браузере введи: http://192.168.1.5/Курсач/login.php.

Примечание: Брандмауэр Windows может блокировать входящие соединения. Если не заходит, временно отключи брандмауэр или добавь порт 80 в исключения.

Что нужно проверить, если сайт не работает:

Права доступа: Файлы data_storage.json и my_mail_log.txt должны быть доступны для записи (на Windows обычно проблем нет, на Linux/Mac нужно выставить права 777).

Версия PHP: Убедись, что на новом сервере стоит PHP версии 7.4 или выше.

Имена таблиц: MySQL в Windows не чувствителен к регистру (Products и products — одно и то же), но в Linux это разные таблицы. Пиши названия таблиц всегда маленькими буквами, как в коде.
